# Model Lab v2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 Model Lab 升级为完整调优工作台：本地 LLM 任意角色、TTS 音色克隆与音色库、preset 预设体系、带延迟/A-B/直录/全链路的预览闭环。

**Design doc:** `docs/superpowers/specs/2026-07-17-model-lab-v2-design.md`（各任务的字段名、API 形状、存储路径以设计文档为准）。

**Tech Stack:** FastAPI + Pipecat `run_test()` 预览、JSON 文件存储（`model_presets.json` / `models/voices/`）、React ModelLabScreen 动态 schema 渲染。

## Global Constraints

- 预览必须复用真实 builder 函数，禁止 mock 服务（既有原则）。
- values/preset 存储保持 schema-free + forward-compat（未知 key 保留不删）。
- 不改 WebRTC 与 pipeline 形状；配置仍是"下次连接生效"。
- 新增存储文件全部 gitignore，不提交任何密钥/音频。
- 每个任务先写失败测试再实现（仓库既有 TDD 约定）。

> **Step 0 验证结论（2026-07-18，对着运行中的 oMLX 实测）：**
> `ref_audio` 必须是 **base64 编码的 WAV 字节**（传文件路径被拒：`Invalid base64 encoding in 'ref_audio' field`），`ref_text` 为该参考音频的逐字文本，两者均为可空 string。
> **已知阻塞：** 当前安装的 oMLX.app 内置 `mlx_audio` 版本损坏，任何带 ref_audio 的请求（16k/24k/44.1k/48k 采样率均试过）都返回 500：`cannot import name 'resample_audio' from 'mlx_audio.utils'`。本应用侧契约已按验证结论正确接线并将该错误干净透传；升级 oMLX 后 Step 4 听感验收即可补做，应用代码无需改动。

## 验收总览

1. Text tab 本地 adapter 可编辑 persona，并可在预设间一键切换（内置 5 种 + 自定义）。
2. Voice tab 可上传/录制参考音频建音色，voice 下拉可选，预览与正式通话均生效。
3. 每次预览显示耗时；TTS 可保留多条历史来回对比。
4. Listening tab 可直接录音测试。
5. 全链路 tab 一次录音看到 STT/LLM/TTS 分段结果与耗时。
6. `npm run build`（client/）与 `uv run python -m unittest discover -s tests` 全绿；新增存储文件均不入库。
